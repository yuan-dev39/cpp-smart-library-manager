# システム設計書

## 1. 要件分析

本システムは小規模な図書室を対象とし、紙による管理で発生しやすい記入ミス、在庫状況の把握しにくさ、検索の手間を解決します。主な利用場面は、図書情報の管理、貸出・返却、条件検索、在庫統計、ローカルファイルへの保存です。

## 2. アーキテクチャ

次の三層構成を採用しています。

1. データモデル層：`Book`が図書番号、書名、著者、在庫状況を保持します。
2. 業務ロジック層：`Library`が検証、追加、変更、削除、検索、貸出、返却、統計、保存を担当します。
3. 表示層：`main.cpp`はメニュー表示、入力解析、結果出力のみを担当します。

入出力と業務規則を一つの大きなクラスに混在させないことで、コアロジックをGUIなどから再利用でき、単体テストも容易になります。

## 3. 業務上の制約

- 図書番号は正の整数であり、重複してはいけません。
- 書名と著者名は空にできません。
- 総冊数に負の値は指定できません。
- 貸出中冊数は0以上かつ総冊数以下である必要があります。
- 在庫がない場合は追加貸出できません。
- 貸出記録がない図書は返却できません。
- 貸出中の図書は削除できません。
- 総冊数を変更する際、現在の貸出中冊数より小さい値にはできません。

## 4. データ保存

データはヘッダー付きUTF-8 CSVとして保存します。書き込み時にカンマ、引用符、改行を適切にエスケープし、読み込み時に項目数、整数形式、図書番号の重複、業務規則を検証します。保存時は先に`.tmp`一時ファイルへ書き込み、成功後に正式ファイルと置き換えることで、書き込み中断による破損リスクを減らします。

## 5. アルゴリズムと計算量

授業課題および小規模図書室に適した`std::vector<Book>`を使用します。

| 操作 | 計算量 |
| --- | --- |
| 図書番号検索 | O(n) |
| 部分一致検索 | O(n) |
| 追加後の並べ替え | O(n log n) |
| 削除 | O(n) |
| 統計 | O(n) |

データ量が増えた場合は`std::unordered_map<int, size_t>`による索引を追加し、図書番号検索を平均O(1)へ改善できます。

## 6. テスト設計

自動テストでは、正常な追加、図書番号の重複拒否、番号順の並べ替え、書名・著者検索、在庫切れ、過剰貸出、貸出中の削除防止、不正な総冊数変更、返却、統計、およびカンマを含むCSVデータの保存・再読込を検証します。
